# Report Limitations — Decision-Support, Not Diagnosis

The report generated by this project is produced from automated segmentation
statistics (organ coverage, model confidence, model entropy) and, when RAG
grounding is enabled, a small set of retrieved reference passages about normal
anatomy and imaging conventions. It is written in a clinical-sounding style
for readability, but several things distinguish it from an actual diagnostic
radiology report:

- It is generated from a single 2D slice, not a full clinical study, patient
  history, or radiologist review.
- The underlying segmentation model has known, measured limitations (see this
  project's published per-organ accuracy and failure analysis) — it is not a
  clinically validated or regulatory-cleared diagnostic device.
- The reference passages used for grounding describe general anatomy and
  imaging concepts; they are not patient-specific medical advice and cannot
  substitute for professional interpretation of an individual's imaging study.
- Any statement in the report that an organ or region "requires human review"
  reflects a measured uncertainty signal (low confidence or high entropy) from
  the model, not a clinical judgment that something is wrong.

This project is a research and education demonstration. It is not a medical
device and is not intended for diagnostic use. Any real clinical decision
should rely on a qualified healthcare professional's review of the complete
imaging study and clinical context, not on this tool's output alone.
